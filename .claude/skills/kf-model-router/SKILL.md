# kf-model-router — 省原则：模型智能路由

> **核心原则**：最好的模型和最性价比的模型搭配，结合工具方法稳固 ROI。
> **管理原则**：想的美，做的实。计划用极致模式（opus），执行用普通模式(sonnet)。

---

## 模型路由规则

| 阶段 | 模型 | 原因 |
|------|------|------|
| **计划/设计** | Opus（极致模式） | 需要深度推理、架构决策、权衡取舍 |
| **执行/编码** | Sonnet（普通模式） | 效率优先，常规编码任务不需要极致推理 |
| **代码审查** | Sonnet（普通模式） | 模式匹配为主，性价比高 |
| **文档生成** | Sonnet（普通模式） | 结构化输出，低成本 |
| **Bug 排查** | Opus（极致模式） | 需要深度上下文理解和推理链 |
| **简单问答** | Haiku（轻量模式） | 极低成本，快速响应 |

---

## 使用方式

### 自动路由

在 Claude Code 中根据任务类型自动选择模型：

```
/set-model opus    # 切换到极致模式（计划/设计/排查）
/set-model sonnet  # 切换到普通模式（执行/审查/文档）
/set-model haiku   # 切换到轻量模式（简单问答）
```

### 手动触发

```
模型路由        # AI 分析当前任务并推荐模型
省模式          # 自动进入执行模式（sonnet）
极致模式        # 自动进入计划模式（opus）
```

## ROI 参考

| 模型 | 相对成本 | 适用场景 |
|------|---------|---------|
| Opus | 100% | 架构设计、复杂 Bug、需求澄清 |
| Sonnet | ~33% | 日常编码、代码审查、文档 |
| Haiku | ~10% | 简单问答、格式转换、信息提取 |

**建议配比**：Opus 20% + Sonnet 70% + Haiku 10%，综合成本 ~50%。
